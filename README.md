# ДомСфера — сеть магазинов товаров для дома

Учебный сайт для производственной (эксплуатационной) практики. Проект разработан без CMS на PHP, HTML, CSS и JavaScript.

## Быстрый запуск в XAMPP

1. Скопируйте папку проекта в `C:\xampp\htdocs\domsfera`.
2. Запустите Apache в панели XAMPP.
3. Откройте `http://localhost/domsfera/`.
4. Убедитесь, что папка `storage` доступна PHP для записи.

## Запуск встроенным сервером PHP

```bash
php -S 127.0.0.1:8080 router.php
```

Откройте `http://127.0.0.1:8080`.

## Учетные записи

- Администратор: `admin` / `Admin2025!`
- Покупатель: `pokupatel` / `Dom2025!`

После первого входа рекомендуется создать новые учетные записи и заменить демонстрационные пароли.

## Реализованные функции

- каталог из 30 товаров с категориями и поиском;
- 15 статей в трех тематических разделах;
- лента из пяти новостей;
- форма обратной связи и ответы администратора;
- самостоятельная регистрация покупателя;
- создание пользователей администратором;
- роли «Администратор» и «Покупатель»;
- личный кабинет;
- обычный и высококонтрастный стили;
- карта сайта;
- собственная страница ошибки 404;
- адаптивная верстка.

## Docker

```bash
docker build -t dom-sphere .
docker run --rm -p 8080:10000 dom-sphere
```

После запуска сайт доступен по адресу `http://localhost:8080`, проверка состояния — `http://localhost:8080/healthz`.

## Бесплатный деплой на Render

В корне репозитория находится Blueprint `render.yaml`, который создает бесплатный Docker Web Service.

1. В Render выберите **New → Blueprint**.
2. Подключите GitHub-репозиторий `nogot261/dom-sphere`.
3. Render прочитает `render.yaml`; подтвердите создание сервиса `dom-sphere`.
4. После завершения сборки откройте выданный адрес `onrender.com`.

Контейнер слушает значение переменной `PORT`, а Render проверяет путь `/healthz`.

Важно: файловая система бесплатного Web Service временная. Новые регистрации и обращения сохраняются в `storage/*.json`, но могут исчезнуть после перезапуска или нового деплоя. Для постоянного хранения потребуется внешняя база данных или платный persistent disk.

## Структура

- `includes/` — общие функции, шапка и подвал;
- `data/content.php` — товары, статьи и новости;
- `storage/` — учетные записи и обращения;
- `assets/` — стили, скрипты и локальные изображения;
- `tests/` — проверочный сценарий.
